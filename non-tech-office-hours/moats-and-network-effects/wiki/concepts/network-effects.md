# Network Effects

**Category:** Umbrella — NFX taxonomy
**Source:** raw/nfx-network-effects-manual.md

## Definition

A network effect exists when each additional user makes the product more valuable for every existing user. More users → more value per user. This is a structural property of the business, not a growth strategy.

## Mechanism

- Each new node adds value to existing nodes, so the network compounds: value scales faster than headcount.
- The compounding creates a self-reinforcing loop — more value attracts more users, which adds more value.
- Distinguish from viral growth: virality drives acquisition speed; network effects drive retention and value. A product can be viral without any network effect, and vice versa. NFX is explicit: conflating the two is the most common founder mistake.

## When it applies

- Each new user measurably increases value for existing users — not just "more revenue for the company."
- The value increase is felt by users directly (not just inferred by the founder).
- Removing users from the network would degrade the product, not just shrink the market.

## Kill-test

Does each new user actually increase value for existing users? If you turned off the network and left only one user, would the product still work at full value?

## AI-native erosion check

Network effects are generally strengthened in the AI era. Data network effects, in particular, compound faster when ML loops can extract signal from larger datasets. The moat survives the cost-collapse. Caveat: the bar for what counts as a real network effect has risen. LLMs make it easy to build recommendation feeds and personalization systems that look like network effects but are not — they are single-player AI features. Push back hard when a founder conflates "our AI gets smarter with data" with a genuine multi-user network effect.
